# Steam 市场批量上架助手

这是一个用于 Steam Community 页面的 Tampermonkey 脚本，当前版本支持：

- 扫描当前可见库存页物品
- 为每个物品添加独立勾选框
- 查询 Steam 官方市场最低价
- 自动回填价格，并允许手动修改
- 对已选物品统一应用一个价格
- 一键按已确认价格自动执行上架队列
- 一键将当前页可见库存按最低价自动上架
- 跨分页逐页清仓整个库存
- 跨分页按指定统一售价直接上架整个库存，不查询最低价
- 跨分页按最低价减 0.01 元清仓整个库存
- 运行中可暂停/继续
- 详细日志展示每个物品的处理步骤
- 面板在 Steam Community 全站共用同一套位置与大小，可在页面中拖动、缩放并记住布局

## 安装方法

1. 在 Chrome 或 Edge 安装 Tampermonkey。
2. 打开 Tampermonkey 控制面板。
3. 新建脚本。
4. 将 `steam-market-batch-seller.user.js` 的内容完整粘贴进去。
5. 保存并确认脚本已启用。

## 生效页面

- `https://steamcommunity.com/*`

面板会在 Steam Community 页面中显示，库存页和非库存页使用的是同一个面板布局；库存相关操作只会在库存页启用。

## 推荐使用流程

1. 打开任意 Steam Community 页面，确认右上角出现 `Steam 批量上架助手` 面板。
2. 进入库存页并等待物品网格加载完成。
3. 如果页面后加载，点击 `扫描物品`。
4. 只勾选你想上架的那些物品，或在确实需要时使用 `全选当前页`。
5. 点击 `查询最低价`。
6. 检查每一行自动带出的价格，必要时手动修改。
7. 点击 `确认价格并自动上架`。
8. 脚本会自动打开出售框、勾选协议、点击同意价格并尝试完成上架。

查价策略当前已改为严格模式：

1. 只接受 Steam `priceoverview` 返回的 `lowest_price`
2. 不再回退到 `median_price`
3. 如果接口没有 `lowest_price`，该物品直接记为查价失败
4. 日志会记录 `lowest_price`、`median_price`、`volume` 和最终解析结果

如果你想直接把当前页可见库存按最低价一口气处理，可以点击 `全页最低价一键上架`。这个按钮会：

1. 扫描当前页可见物品
2. 全选当前页
3. 逐个查询最低价
4. 自动跳过查价失败的物品
5. 对查价成功的物品直接启动自动上架队列

注意：这里的“全页”指当前已加载且当前页可见的库存，不会自动翻页处理整个账号库存。

如果你想从第一页开始把整个库存逐页清掉，可以点击 `跨分页全库存清仓`。这个按钮会：

1. 自动回到库存第一页
2. 逐页扫描当前页可见物品
3. 当前页全选并逐个查询最低价
4. 自动跳过当前页查价失败的物品
5. 把当前页查价成功的物品送进自动上架队列
6. 当前页处理完成后自动翻到下一页，直到没有下一页为止

注意：这里的“全库存”仍然建立在 Steam 当前库存分页控件可正常翻页的前提上；如果 Steam 某些分类或筛选状态只显示部分库存，那么脚本只会处理当前这个视图下的全部分页内容。

如果你想跳过最低价查询，直接用一个指定价格上架整个库存，可以先在 `统一售价` 输入框填入价格，再点击 `全库存指定价直接上架`。这个按钮会：

1. 校验 `统一售价` 是否是有效正数价格
2. 自动回到库存第一页
3. 逐页扫描当前页可见物品
4. 当前页全选并把统一售价写入每个物品
5. 不请求 Steam 最低价接口，直接把当前页物品送进自动上架队列
6. 当前页处理完成后自动翻到下一页，直到没有下一页为止

如果你想把整个库存按 `最低价 - 0.01` 的策略逐页清掉，可以点击 `全库存最低价-0.01上架`。这个按钮会：

1. 自动回到库存第一页
2. 逐页扫描当前页可见物品
3. 当前页全选并逐个查询最低价
4. 对查价成功的物品统一改成 `lowest_price - 0.01`
5. 自动跳过查价失败或调价失败的物品
6. 把当前页可上架物品送进自动上架队列
7. 当前页处理完成后自动翻到下一页，直到没有下一页为止

价格下限当前按 `0.01` 保护，不会生成非正数价格。

运行过程中可以点击 `暂停脚本`。暂停是任务级暂停：当前这个小步骤结束后停止推进，点击 `继续脚本` 后再接着跑。

## 安全说明

- 脚本会自动勾选协议并自动点击出售流程按钮，请先小批量测试。
- Steam 页面结构可能变动，若按钮或输入框失效，需要再调整选择器。
- 如果你的账号启用了移动令牌或挂单确认限制，Steam 端仍可能产生待确认状态。

## 快速验收

1. 打开一个非库存的 Steam Community 页面，确认面板显示且库存操作按钮不可用。
2. 打开库存页，确认同一面板位置被沿用。
3. 在任意 Steam Community 页面拖动面板到新位置，再缩放大小；切到库存页或非库存页并刷新后，确认位置与尺寸仍被沿用。
4. 只勾选两件物品，确认 `已选择` 计数只增加到 `2`，不会连带选中整页。
5. 点击 `查询最低价`，确认行内出现价格或失败状态。
6. 选择一件物品执行，确认 Steam 出售框被打开且脚本会自动尝试完成上架流程。
7. 查看日志区，确认能看到“读取详情、打开弹窗、填写价格、点击同意价格、提交结果”等步骤。
